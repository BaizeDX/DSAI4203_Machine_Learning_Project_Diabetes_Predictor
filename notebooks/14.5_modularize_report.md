### 14.5 Report Modularization Note

This document explains how the reorganized notebook series should be translated into a clean report structure.

The updated report should follow the project story in five parts:

1. **Problem framing and exploratory analysis**  
   Use Notebooks 01-03 to introduce the dataset, the baseline context, and the early model-family exploration.

2. **Boosting-model development**  
   Use Notebooks 04-09 to describe the default boosting baselines, the XGBoost refinement path, and the ensemble checkpoint.

3. **Methodological strengthening**  
   Use Notebooks 10, 10.5, 11, and 11.5 to explain why the project moved from raw holdout comparisons toward a more trustworthy evaluation framework.

4. **Validation-design analysis**  
   Use Notebook 12 for the tail-holdout discussion and Notebook 13 for the 5-fold / 7-fold / 10-fold decision. This section should make it clear that validation reliability became a central design question rather than a minor implementation detail.

5. **Final synthesis and presentation assets**  
   Use Notebook 13.5 for the overall experiment summary and Notebook 14 for figures that support the final narrative.

### Recommended Writing Emphasis

- Distinguish exploratory gains from methodologically trustworthy gains.
- Present validation choices as part of the contribution, not merely background setup.
- Keep Notebook 11.5 as an intermediate phase summary and reserve Notebook 13.5 for the full project-level conclusion.
- Use Notebook 14 figures selectively to support the final storyline rather than to reproduce every intermediate plot.

### Practical Outcome

The reorganized structure makes the report easier to read because it now progresses from:

- understanding the data
- exploring candidate methods
- refining the strongest model family
- improving evaluation reliability
- summarizing the final methodology with supporting visuals

That sequence is more coherent for both a written report and a presentation deck.
