### 10.5 Experimental Journey Summary and Improvement ###

------------------Early Experimental Exploration (Experiments 03-10)------------------

In the early stages of the project, we conducted a series of model optimization experiments to gradually improve model performance. These experiments document the complete improvement process but also exposed several methodological issues.

*Experiment 03: XGBoost Default Parameters
In our first attempt at XGBoost, using default parameters (100 trees, learning rate 0.1), the validation set AUC reached 0.7192, a significant improvement compared to the decision tree baseline (0.6825). The learning curve showed a continuous upward trend, indicating the need for more trees.

*Experiments 04-05: Model Capacity Adjustment
Increasing the number of trees to 300 (Experiment 04) improved the AUC to 0.7255. Increasing it to 500 trees and tuning max_depth (Experiment 05) achieved an AUC of 0.7264. The learning curve showed the model was still converging, but the rate of improvement was gradually decreasing.

*Experiments 06-07: Feature Engineering Exploration
Interactive features were created based on medical knowledge (Experiment 06), with an AUC of 0.7251. The original features were mistakenly deleted (Experiment 07), resulting in information loss and no improvement in AUC. This stage revealed that interactive features should not replace original features, but rather serve as a supplement.

*Experiments 08-09: Model Fusion Attempts
LightGBM and XGBoost performed similarly (Experiment 08), with an AUC of 0.7248. A simple fusion of the two models (Experiment 09) slightly increased the AUC to 0.7256. This demonstrates that different models can learn from different perspectives.

------------------Problems with Early Experiments------------------

Although the AUC improved from 0.682 to 0.726, these experiments had some fatal problems, which includes:

*Validation set reuse
Always using a fixed split with `random_state=42`, the same validation set was used in 7 experiments. This may lead to overfitting of the model to this validation set, resulting in overly optimistic evaluation results.

*Improper feature processing
Interacting features were used instead of original features, resulting in the loss of independent information about `age` and `family_history_diabetes`.

*Category encoding risk
Manual one-hot encoding may cause inconsistencies in the number of columns between the training and test sets.

------------------Solutions & Design in Lab11------------------

*5-Fold Cross-Validation
Randomly divide the data into 5 parts, maintaining a consistent class ratio in each part. Alternately use 4 parts for training and 1 part for validation, repeating 5 times. Each sample is validated once to avoid overfitting to a single validation set.

*OOF Evaluation Metric
OOF (Out-Of-Fold) prediction refers to the prediction for each sample coming from a model that has never seen it before. OOF AUC is the most realistic estimate of generalization ability and is more reliable than single-validation AUC.

*Feature Processing Optimization
Retain all original features (age, family_history_diabetes, etc.) while adding interactive features as supplementary features (age_family_history, age_bmi, etc.). Ensure the model can learn both the independent effects and interactions of the original features simultaneously.

*Native Category Support
Use XGBoost's native categorical support instead of manual One-Hot encoding to avoid the problem of inconsistent column counts between the training and test sets, resulting in cleaner code and less memory usage.

*Early Stopping Mechanism
Training is automatically stopped when the validation set AUC does not improve for 100 consecutive rounds. The optimal number of trees is automatically determined to prevent overfitting.
