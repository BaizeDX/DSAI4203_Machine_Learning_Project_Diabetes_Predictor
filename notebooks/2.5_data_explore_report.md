- Data Exploration Summary (01_EDA)

Before building any predictive models, we conducted a comprehensive exploratory data analysis to understand the dataset's characteristics and identify potential challenges.

The dataset contains 700,000 training samples and 300,000 test samples with 26 features, including both numerical and categorical variables. The target variable `diagnosed_diabetes` shows an imbalance: approximately 62.3% of samples are labeled as having diabetes (class 1), while 37.7% are non-diabetic (class 0). This imbalance ratio of 1.65 indicates that the positive class is the majority, which influences model evaluation considerations.

Analysis of numerical features revealed that `family_history_diabetes` shows the strongest positive correlation with the target (0.211), followed by `age` (0.161) and `systolic_bp` (0.107). These three features emerged as the most potentially predictive variables. Outlier detection identified that some features contain extreme values, particularly `physical_activity_minutes_per_week` with 4.78% outliers, suggesting the need for robust preprocessing.

Categorical features include gender, ethnicity, education level, income level, smoking status, and employment status. Initial examination showed that the categorical distributions are reasonably balanced, with no single category dominating excessively.

The exploration confirmed that there are no missing values in the dataset, simplifying the preprocessing pipeline. Feature distributions were examined and found to be generally plausible, with most variables following expected patterns based on medical knowledge.

- Baseline Model Results (02_baseline)

A decision tree classifier was established as the baseline model using default parameters with a maximum depth of 5. The model was evaluated on a 20% holdout validation set, achieving an AUC of 0.6825.

The classification report revealed significant class imbalance issues. The model achieved a recall of 0.88 for the diabetic class but only 0.30 for the non-diabetic class, indicating a strong bias toward predicting the majority class. The precision was 0.68 for diabetic and 0.60 for non-diabetic cases.

Feature importance analysis from the decision tree showed that only three features contributed meaningfully to predictions: `family_history_diabetes` (44.8%), `physical_activity_minutes_per_week` (29.9%), and `age` (21.8%). The remaining features had negligible importance, suggesting that the shallow tree structure was insufficient to capture complex relationships in the data.

These baseline results established a reference point for subsequent model improvements and highlighted two immediate priorities: addressing the class imbalance issue and utilizing more sophisticated models capable of leveraging the full feature set.

- Further model choice

The baseline decision tree's limitations—particularly its inability to effectively utilize most features and its strong bias toward the majority class—motivated the transition to XGBoost. XGBoost's gradient boosting framework is better suited to capture complex feature interactions and can incorporate class weighting to address imbalance. The following experiments systematically explore XGBoost's capabilities, starting with default parameters and progressively tuning model complexity, adding engineered features, and refining the evaluation methodology.
